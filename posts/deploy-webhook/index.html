使用 Webhook 实现博客自动部署

Webhook 的具体实现，以及如何使用 Webhook 实现博客的自动部署。

2024.06.05
阅读时长: 3 分钟

什么是 Webhook

顾名思义 hook 即为钩子。就像钓鱼一样，我们如何判断是否钓到了鱼，自然就是看鱼钩有没有触动，触动了我们就需要做出拉鱼竿的操作，来作为鱼上钩的反应动作。Webhook 就是相当于一个钩子，当有事件发生时，Webhook 会触发相应的动作。

实现博客自动部署理想方案

先是将博客的静态资源托管到 GitHub 上，再在服务器上使用 git clone 命令将仓库克隆到服务器，配置好 Nginx，这样我们的博客就部署好了。但是在我们以后更新博客的时候，每次还需要手动去服务器上 git pull 拉取最新的代码，非常的麻烦。那么我们就可以在服务器上部署好 Webhook，然后将在 GitHub 配置好 Webhook，每当我们提交代码时，Webhook 就会自动触发，然后服务器上的 Webhook 会自动拉取最新的代码，这就完成了博客部署可持续化。

具体实现

这里以将使用 Node.js 和 acme.sh 配置 HTTPS 为例。服务器环境为 Debian 12。

1. 安装 Node.js 和 NPM

首先，确保你的 Debian 系统是最新的：

sudo apt update
sudo apt upgrade

然后，安装 Node.js 和 NPM：

sudo apt install nodejs npm

验证安装是否成功：

node -v
npm -v

2. 设置 Node.js 应用程序

创建一个新的目录来存放你的 Webhook 处理代码：

mkdir ~/webhook-app
cd ~/webhook-app

初始化一个新的 Node.js 项目：

npm init -y

安装 Express 和 body-parser：

npm install express body-parser

在项目根目录下创建一个 index.js 文件并添加以下代码：

const express = require('express');
const bodyParser = require('body-parser');
const { exec } = require('child_process');
const fs = require('fs');
const https = require('https');
const app = express();

const crypto = require('crypto');

const secret = 'yoursecretkey'; // 修改为你想设置的密码

function verifySignature(req, res, buf) {
    const signature = req.headers['x-hub-signature'];
    const hash = `sha1=${crypto.createHmac('sha1', secret).update(buf).digest('hex')}`;

    if (signature !== hash) {
        throw new Error('Invalid signature');
    }
}

app.use(bodyParser.json({ verify: verifySignature }));

const REPO_PATH = '/root/my-repo'; // 修改为你的仓库路径
const GIT_COMMAND = `cd ${REPO_PATH} && git pull`;

app.post('/webhook', (req, res) => {
    const payload = req.body;
    console.log('Received Webhook:', payload);

    // 执行 git pull 命令
    exec(GIT_COMMAND, (error, stdout, stderr) => {
        if (error) {
            console.error(`Error executing command: ${error}`);
            return res.status(500).send('Internal Server Error');
        }

        console.log(`stdout: ${stdout}`);
        console.error(`stderr: ${stderr}`);
        res.status(200).send('Webhook received successfully');
    });
});

const PORT = process.env.PORT || 3000;
const HTTPS_PORT = process.env.HTTPS_PORT || 3443;

// Load SSL certificates
// 修改为你的证书路径，可先不填，等后面使用 acme.sh 获取 SSL 证书后再填
// const privateKey = fs.readFileSync('/etc/nginx/ssl/yourdomain.com.key', 'utf8');
// const certificate = fs.readFileSync('/etc/nginx/ssl/yourdomain.com.crt', 'utf8');

const credentials = { key: privateKey, cert: certificate };

// Start HTTP server
app.listen(PORT, () => {
    console.log(`HTTP Server is running on port ${PORT}`);
});

// Start HTTPS server
https.createServer(credentials, app).listen(HTTPS_PORT, () => {
    console.log(`HTTPS Server is running on port ${HTTPS_PORT}`);
});

3. 安装 acme.sh 并获取 SSL 证书

安装 acme.sh：（建议在 root 用户下进行：先执行 sudo su，然后再执行 cd ~）

curl https://get.acme.sh | sh

加载 acme.sh 环境：

source ~/.bashrc

或者在当前 shell 会话中加载：

~/.acme.sh/acme.sh --upgrade --auto-upgrade

使用 acme.sh 获取 SSL 证书：

~/.acme.sh/acme.sh --issue --standalone -d yourdomain.com -d www.yourdomain.com

将证书安装到指定目录，例如 /etc/nginx/ssl：

mkdir -p /etc/nginx/ssl
~/.acme.sh/acme.sh --install-cert -d yourdomain.com \
    --key-file /etc/nginx/ssl/yourdomain.com.key \
    --fullchain-file /etc/nginx/ssl/yourdomain.com.crt

4. 配置 Nginx 反向代理

安装 Nginx：

sudo apt install nginx

配置 Nginx 以代理传输到你的 Node.js 应用程序。编辑或创建一个新的 Nginx 配置文件：

sudo nano /etc/nginx/sites-available/webhook

添加以下内容：

server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    listen 443 ssl;
    ssl_certificate /etc/nginx/ssl/yourdomain.com.crt;
    ssl_certificate_key /etc/nginx/ssl/yourdomain.com.key;
}

启用该配置并重新启动 Nginx：

sudo ln -s /etc/nginx/sites-available/webhook /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

5. 启动 Node.js 应用

安装 PM2 以管理你的 Node.js 应用程序：

sudo npm install -g pm2

启动你的应用程序并让 PM2 进行管理：

pm2 start index.js
pm2 startup
pm2 save

6. 配置自动续期

acme.sh 会自动续期证书。你可以通过 crontab 确保 acme.sh 自动运行续期：

crontab -e

添加以下行确保 acme.sh 自动运行续期：

0 0 * * * "~/.acme.sh"/acme.sh --cron --home "~/.acme.sh" > /dev/null

7. 在 GitHub 仓库配置 Webhook

先进入 GitHub 仓库的设置页面，选择 Webhooks，然后点击 Add Webhook。，将 Webhook URL 设置为 https://yourdomain.com/webhook，并确保选择 application/json 作为内容类型。 Secret 填入 index.js 中定义的 SECRET，选择 Just the push event，然后点击 Add Webhook。

8. 测试 Webhook

在你第一次添加 Webhook 的时候，GitHub 会发送一个测试事件。你可以在通过进入 GitHub 仓库的 Webhook 设置页面，选择 Recent Deliveries 查看测试事件。在 Response 列中，你可以看到 Body 中的 Webhook received successfully。

结语

到这里，你的 Webhook 已经配置完成。你可以通过向 GitHub 仓库推送代码来触发 Webhook，然后服务器的 Webhook 会自动拉取最新的代码。这样子博客的自动部署就完成了。
